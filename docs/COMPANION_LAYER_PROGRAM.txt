P10 — ENFORCEMENT GATE (the loop becomes mandatory)  [no live model]
--------------------------------------------------------------------------------
BUILD: a Claude/Cursor Stop-hook + PostToolUse hook (shipped in .claude/settings.json
and a generated per-client equivalent) that greps the proof ledger for a FRESH,
anchored, PASSING operate:run entry whose filesChanged/HEAD matches the current
diff, and exits non-zero (blocks turn completion) when absent. Default-install the
pre-commit guard on `sage init`. operate already gates whole-repo test health when
there are no impacted tests (P11 fix landed).
TESTING INFRA (non-fakeable): an enforcement test that runs the Stop-hook against
(a) a repo with a fresh matching ledger entry -> exit 0, (b) a repo whose diff
changed AFTER the proof -> exit non-zero. Assert the hook is WIRED in the shipped
settings.json, not merely present as a file.

P12 — CROSS-MODEL ADAPTER + PER-CLIENT CONTRACT GENERATOR  [proof needs a live non-Claude model]
--------------------------------------------------------------------------------

TESTING INFRA (non-fakeable): a cross-model adapter test that runs the full
operate loop under a NON-Claude agent STUB and asserts the SAME proof artifacts
(anchored ledger entry, same gate set, same proofGraph shape) — proving the
contract is provider-neutral. A contract-parity test asserting every rendered
client file derives from the one source (hash/section check). MARK: the headline
acceptance run needs a live non-Claude model to regenerate a live-repair-corpus
with model != "claude" and passRate > 0.

--------------------------------------------------------------------------------
P13 — LONG-HORIZON AUTONOMOUS DRIVER (the Karpathy loop)  [real run needs a live model]
--------------------------------------------------------------------------------
BUILD: driveGoal(objective): (a) decompose one objective into an ordered/parallel
task DAG, (b) drive each task through runOperate, (c) feed verified output forward /
enqueue follow-ons, (d) loop with a bounded iteration budget + typed stop/approval
until whole-goal acceptance passes. Wire it to the daemon queue so it runs
unattended.
TESTING INFRA (non-fakeable): an autonomous-driver test that drives a 3-task
fixture goal to completion and asserts: each task has its own anchored proof in the
ledger; the run STOPS on a typed condition (goal-complete | budget-exhausted |
needs-approval, never silently); a deliberately-unsatisfiable 4th task yields a
typed blocked, not a false success. A determinism guard: re-running on the
completed fixture is a no-op (idempotent). MARK: "drive a real bug-fix goal to
green" acceptance needs a live model; the DAG/decomposition/stop plumbing is
testable with a stub.
